## Component Descriptions

### Core Files

#### `src/ApiDocService.php` 🎯
Main orchestration service that:
- Initializes the route collector
- Builds the OpenAPI specification
- Handles version and metadata

#### `src/ApiDocServiceProvider.php` 🔧
Laravel service provider that:
- Registers the package routes
- Binds services to the container
- Publishes configuration files

### Collectors

#### `src/Collectors/RouteCollector.php` 📝
Scans Laravel application routes and:
- Filters for API routes (routes with `api/` prefix)
- Extracts HTTP method information
- Collects route URI and parameters
- Passes data to generators

### Extractors

#### `src/Extractors/DtoExtractor.php` 📊
Analyzes controller methods and:
- Extracts Spatie Laravel Data DTO information
- Generates JSON Schema representations
- Documents request/response structure
- Builds parameter documentation

### Generators

#### `src/Generators/OpenApiGenerator.php` 🔄
Generates OpenAPI 3.1.0 specification by:
- Collecting routes via RouteCollector
- Extracting schemas via DtoExtractor
- Formatting specification according to OpenAPI standard
- Including server information

### HTTP Layer

#### `src/Http/Controllers/ApiDocController.php` 🎮
Handles HTTP requests to `/docs` endpoint by:
- Instantiating OpenApiGenerator
- Generating API specification
- Rendering the Blade view with spec data
- Passing configuration to frontend

### Frontend

#### `resources/views/docs.blade.php` ✨
Interactive documentation interface featuring:
- **JavaScript Framework** - Vanilla JS for maximum compatibility
- **Styling** - Tailwind CSS for modern UI
- **Syntax Highlighting** - Highlight.js for code display
- **Features**:
  - Sidebar navigation with method badges
  - Request builder modal
  - Query parameter management
  - Header management
  - Request body editor
  - Real-time response display
  - Status code and timing metrics

### Configuration

#### `config/sculpt.php` 🔧
Configuration options for:
- Documentation title and description
- Version number
- Route prefix (default: `docs`)
- UI theme settings
- Feature toggles

### Routes

#### `routes/api-doc.php` 🛣️
Registers the documentation route:
- Single GET route to display documentation
- Route name: `sculpt.docs.index`
- Configurable prefix

## Configuration Hierarchy

1. **Environment Variables** (highest priority)
   - `SCULPT_APIDOC_ENABLED`
   - `SCULPT_APIDOC_TITLE`
   - `SCULPT_APIDOC_DESCRIPTION`
   - `SCULPT_APIDOC_VERSION`

2. **config/sculpt.php**
   - Published configuration file
   - Overrides defaults

3. **Hardcoded Defaults** (lowest priority)
   - Built-in fallback values

## Extension Points

### Creating Custom Collector

```php
class CustomCollector implements CollectorInterface
{
    public function collect(): array
    {
        // Your custom route collection logic
        return ['paths' => []];
    }
}
```

### Customizing Documentation View

```bash
php artisan vendor:publish --provider="Sculpt\ApiDoc\ApiDocServiceProvider" --tag=views
```

## Performance Considerations

- **Route Scanning**: Only executed when docs are requested (lazy loading)
- **Caching**: Implement caching for large API collections
- **DOM Rendering**: Uses native DOM APIs for optimal performance
- **Network**: Fetch API for cross-origin requests with proper CORS handling

---

## Security Considerations

- CORS must be properly configured for API requests
- Authorization headers can be manually added
- No credentials stored in browser localStorage
- API requests go directly from browser to API server
